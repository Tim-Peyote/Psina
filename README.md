# 🐾 Псина

Умный Telegram-бот с **долговременной модульной памятью**, **системой скиллов** и живым характером.

Не бот-помощник, а **живой участник чата** — слушает, запоминает, отвечает когда зовут, молчит когда не просят. И может стать кем угодно: от Game Master'а до того, кем ты его попросишь быть.

---

## Что умеет

### Память и контекст

- **Помнит людей** — кто что любит, где работает, чем увлекается. Факты извлекаются автоматически из разговоров через LLM
- **Персональные профили на чат** — один и тот же пользователь имеет **отдельный профиль в каждом чате**. Никакого смешивания
- **4 слоя памяти**:
  1. **Context** (оперативка) — последние сообщения, активные сессии
  2. **Transcript** (лог) — все сообщения в БД, не грузятся в LLM целиком
  3. **Structured memory** — факты, предпочтения, связи, события с confidence и relevance
  4. **Vector memory** — семантический поиск через pgvector (embeddings)
- **Compaction** — старые диалоги сжимаются в summary, оригиналы хранятся в БД
- **Forgetting** — relevance decay, TTL для слабых записей, дедупликация, лимиты
- **Контекст-пак для LLM** — строго ограниченный набор данных, жёсткий лимит токенов
- **Полная изоляция чатов** — память чата А никогда не попадёт в чат Б

### Скиллы

- **Модульная система** — устанавливай, удаляй, переключай
- **Автоматическая активация** — Псина сама понимает когда нужен скилл
- **LLM-роутинг** — если нет явного триггера, бот классифицирует намерение через LLM
- **Изолированное состояние** — каждый скилл хранит состояние отдельно для каждого чата
- **Персистентность** — состояние сохраняется между перезапусками

### Встроенные скиллы

#### 🎲 RPG Game Master

Превращает Псину в полноценного Game Master'а для текстовых RPG:

- **Сессия Ноль** — пошаговое создание кампании (мир → фракции → персонаж → система → границы)
- **D20 / PbtA / D100 / Freeform** — поддержка любых систем
- **Броски костей** — `!roll 1d20+5`, `!roll pbta+2`, advantage/disadvantage
- **Лист персонажа** — раса, класс, уровень, HP, статы, инвентарь, статус-эффекты
- **Нарративные техники** — «Да, но...», «Загруженный вопрос», часы эскалации, последовательные раны
- **Боевая система** — инициатива, урон, криты (Nat 20), провалы (Nat 1)
- **NPC** — с мотивациями, памятью взаимодействий, характерами
- **Журнал кампании** — лог ключевых событий
- **Управление состоянием** — `!flag`, `!clock`, `!hp`, `!add`, `!rest`

### Поведение

- **Живой характер** — верный, честный, с юмором. Не подлизывается, может огрызнуться, не терпит издевательств
- **Подстраивается под чат** — формальный стиль или неформальный, адаптируется под вайб группы
- **Цензура настраивается** — строгий, умеренный или свободный уровень
- **Реакции на сообщения** — ставит 👍 ❤️ 😂 👀 🔥 когда есть повод
- **Анти-хаос** — кулдауны, лимиты ответов в час, детекция эскалации
- **Детекция агрессии** — авто-молчание при издевательствах
- **Дневные сводки** — краткий пересказ событий за день
- **Напоминания** — «напомни завтра в 15 что совещание» или «напомни @Васе завтра в 3»
- **Поиск в интернете** — DuckDuckGo, кэширование, лимиты

## Как работает

### Основной пайплайн

```
Сообщение → Контекст → Память → Факты → Связи → Скилл-роутинг
→ (скилл?) → да: handler → ответ
→ нет: Маршрутизация → Проверка уверенности → Анализ знаний → Память → Контекст-пак → LLM → ответ
```

### Пайплайн скиллов

```
Сообщение → Skill Router
├── Команда? (/rpg) → Активировать скилл → handler.process_message()
├── Активная сессия? → Продолжить скилл → handler.process_message()
├── Триггер-слово? → Активировать скилл → handler.process_message()
├── LLM классификация? → Если совпало → handler.process_message()
└── Нет → Обычный пайплайн Псины
```

### Пайплайн памяти

```
Лог (БД) → Extraction (батчи) → Structured Memory (факты с embeddings)
→ Retrieval (vector + keyword) → Context Pack (лимитированный) → LLM
```

Псина **читает все сообщения**, но **отвечает только** когда:
- Позвали по имени (Псина, пёс, пёсик)
- Ответили на её сообщение
- Находятся в активной сессии диалога
- Активирован скилл

Остальное — слушает и запоминает.

## Гибкость и расширяемость

### Как добавить новый скилл

Скиллы следуют стандарту **SKILL.md** — это markdown-файл с YAML frontmatter, который описывает имя, описание и инструкции.

Система поддерживает **два режима исполнения**:

#### PromptOnlySkill — только инструкции
Для простых скиллов достаточно одного `SKILL.md`. Бот загрузит инструкции и передаст их LLM.

```
src/skills/my_skill/
└── SKILL.md          # имя, описание, инструкции
# handler.py НЕ НУЖЕН — используется дефолтный
```

#### ExecutableSkill — инструкции + кастомная логика
Если скиллу нужна своя логика (команды, броски, API, управление состоянием) — добавь `handler.py`.

```
src/skills/my_skill/
├── SKILL.md          # имя, описание, инструкции
└── handler.py        # async def process_message(msg, chat_id, user_id) -> str
```

**SKILL.md** содержит:
```yaml
---
name: my-skill
description: Краткое описание (до 1024 символов) — по нему бот понимает когда использовать скилл
license: MIT
---
# Instructions
Подробные правила поведения скилла...
```

**Как это работает (Progressive Disclosure):**
1. **Discovery** — при старте читается только `name` + `description` всех скиллов (~50-100 токенов каждый)
2. **Activation** — когда задача совпадает с `description`, загружается полный SKILL.md (~5000 токенов)
3. **Execution** — если есть `handler.py` — вызывается он (ExecutableSkill), иначе — дефолтный handler (PromptOnlySkill)

**Архитектура скиллов:**
- **skill = инструкция/правила** (SKILL.md)
- **tool = реальное действие** (dice.py, web_search, memory API, state_manager)
- **handler = исполнитель** (кастомный для ExecutableSkill, дефолтный для PromptOnlySkill)

**Главный принцип:** LLM не хранит знания внутри себя — она получает их из подключаемых источников по мере необходимости.

### Примеры скиллов

| Скилл | Режим | Зачем свой handler |
|---|---|---|
| **RPG Game Master** | ExecutableSkill | Броски костей, команды (!roll, !hp), Session Zero, управление состоянием |
| **Переводчик** | PromptOnlySkill | Достаточно инструкций «ты переводчик EN→RU» |
| **Поэт** | PromptOnlySkill | Достаточно инструкций «ты пишешь стихи» |
| **Погода** | ExecutableSkill | Вызов внешнего API, парсинг ответа |

### Что уже готово для расширения

- **Реестр скиллов** — установка, удаление, вкл/выкл через API и код
- **State Manager** — CRUD JSON-состояния на `(skill_slug, chat_id)`
- **Skill Router** — 4 уровня уверенности: команда (1.0) → сессия (0.8) → триггер (0.6) → LLM (0.5)
- **Event Log** — аудит-лог всех действий скиллов
- **Admin API** — `/api/skills` для управления

## Быстрый старт

```bash
# 1. Клон
git clone https://github.com/Tim-Peyote/Psina.git
cd Psina

# 2. Конфиг
cp .env.example .env
# Заполни TELEGRAM_BOT_TOKEN и LLM_API_KEY

# 3. Запуск
docker compose up -d --build

# 4. Миграции
docker compose exec bot alembic upgrade head
```

### Локально

```bash
cp .env.example .env
docker compose up -d db redis
pip install -e .
DATABASE_URL_SYNC=postgresql+psycopg2://postgres:postgres@localhost:5432/zalutka \
  alembic upgrade head
python -m src.main
```

### Без LLM (mock режим)

```env
LLM_PROVIDER=mock
```

Бот будет работать, но отвечать заглушками. Для полноценной работы нужен Qwen или другой LLM.

### Обновление

```bash
git pull
docker compose up -d --build
docker compose exec bot alembic upgrade head
```

## Безопасность

- `db` и `redis` не опубликованы наружу
- `admin_api` привязана к `127.0.0.1:8000`
- Для внешнего доступа — reverse proxy (`nginx`/`caddy`) с HTTPS и авторизацией
- В продакшене замени `ADMIN_API_SECRET` на длинное случайное значение
