# 🐾 Псина

Умный Telegram-бот с **долговременной модульной памятью**, **системой скиллов** и живым характером.

Не бот-помощник, а **живой участник чата** — слушает, запоминает, отвечает когда зовут, молчит когда не просят. Может стать кем угодно: от Game Master'а до напоминалки или записной книжки.

---

## Что умеет

### Память и контекст

- **Помнит людей** — кто что любит, где работает, чем увлекается. Факты извлекаются автоматически через LLM
- **Персональные профили на чат** — один и тот же пользователь имеет **отдельный профиль в каждом чате**. Никакого смешивания
- **Тиерованная память** — три уровня важности с разным TTL:
  - Постоянные факты (профессия, город, семья) — `confidence ≥ 0.8`, без TTL
  - Полупостоянные (текущий проект, предпочтения) — `confidence ≥ 0.65`, 30 дней
  - Временные (планы на неделю, текущая задача) — `confidence ≥ 0.6`, 7 дней
- **4 слоя памяти**:
  1. **Context** (оперативка) — последние сообщения, активные сессии
  2. **Transcript** (лог) — все сообщения в БД
  3. **Structured memory** — факты, предпочтения, связи, события с confidence и relevance
  4. **Vector memory** — семантический поиск через pgvector (embeddings)
- **Compaction** — старые диалоги сжимаются в summary
- **Forgetting** — relevance decay, TTL, дедупликация (инкремент frequency вместо дублей), лимиты
- **Полная изоляция чатов** — память чата А никогда не попадёт в чат Б

### Скиллы

- **Модульная система** — устанавливай, удаляй, переключай
- **Автоматическая активация** — Псина сама понимает когда нужен скилл, без команд
- **LLM-роутинг** — классифицирует намерение через LLM если нет явного триггера
- **Изолированное состояние** — каждый скилл хранит состояние отдельно для каждого чата
- **Атомарное состояние** — per-(skill, chat) блокировки предотвращают гонки при одновременных сообщениях

### Встроенные скиллы

#### 🎲 RPG Game Master (`agent_rpg`)

Полноценный Game Master для текстовых RPG. **Управление через естественный язык** — никаких команд.

- **Мульти-сессии** — несколько параллельных кампаний в одном чате (до 5)
- **Список игроков** — только зарегистрированные участники попадают в игру. Остальные получают обычные ответы бота
- **Переключение сессий** — «переключись на Cyberpunk», «покажи все игры»
- **Сессия Ноль** — пошаговое создание кампании (мир → фракции → персонаж → система → границы)
- **LLM-обогащение мира** — после каждого шага бот генерирует атмосферу, NPC, фракции
- **Вступительная сцена** — после Session Zero сразу начинается игра с яркой opening scene
- **D20 / PbtA / D100 / Freeform** — поддержка любых систем
- **Броски костей** — `!roll 1d20+5`, `!roll pbta+2`, advantage/disadvantage
- **Лист персонажа** — раса, класс, уровень, HP, статы, инвентарь, статус-эффекты
- **Боевая система** — инициатива, урон, криты (Nat 20), провалы (Nat 1)
- **NPC** — с мотивациями, памятью взаимодействий
- **Журнал кампании** — лог ключевых событий

**Управление голосом:**
- «начни новую игру / давай в DnD» → создать сессию (Session Zero)
- «продолжи / где мы остановились» → вернуться к активной игре
- «добавь @Васю / хочу играть» → добавить игрока
- «пауза / перерыв» → поставить на паузу
- «заверши игру» → закончить кампанию
- «покажи все игры» → список сессий

#### 📝 Заметки (`agent_notes`)

Записная книжка чата. Все заметки изолированы по `chat_id`.

- **Естественный язык** — «запомни что завтра встреча», «покажи мои заметки», «удали заметку про кофе»
- **CRUD** — создание, поиск, обновление, удаление
- **Теги** — автоматически извлекаются LLM-ом
- **Stateless** — не занимает активную сессию, работает как разовый вызов

#### ⏰ Напоминания (`agent_reminders`)

Управление напоминаниями через естественный язык.

- **LLM-парсинг времени** — «через 30 минут», «завтра в 9», «в пятницу в 15:00»
- **Для других людей** — «напомни @Васе завтра в 3 что встреча»
- **Список и отмена** — «мои напоминания», «отмени напоминание про звонок»
- **Stateless** — работает поверх таблицы `reminders`, не требует активной сессии

### Поведение

- **Живой характер** — НЕ ИИ, НЕ бот, НЕ модель. Никогда не раскрывает техническую природу
- **Адаптивный** — подстраивается под вайб чата: в неформальном шутит жёстче, в формальном — серьёзнее
- **Знает текущую дату** — дата и время МСК передаются во все LLM-вызовы, модель не путает год
- **Правильные теги** — `@username` берётся из реального профиля пользователя в этом чате, имена изолированы по `chat_id`
- **Цензура настраивается** — строгий, умеренный или свободный уровень
- **Реакции на сообщения** — 👍 ❤️ 😂 👀 🔥 💩 😤 🥺 🐾 — выбирает по контексту
- **Анти-хаос** — кулдауны, лимиты ответов, детекция эскалации
- **Детекция агрессии** — авто-молчание при издевательствах
- **Дневные сводки** — краткий пересказ событий за день
- **Поиск в интернете** — DuckDuckGo, структурированный вывод с источниками, определение типа запроса (цена/новость/инструкция/факты)
- **Проактивные сообщения** — утренние приветствия в группах (8-10 утра) с кулдаунами
- **Гибкие режимы** — `observer` | `assistant` | `social` | `game_master`

## Как работает

### Основной пайплайн

```
Сообщение → Контекст → Память → Факты → Связи → Скилл-роутинг
→ (скилл?) → да: handler → ответ
→ нет: Маршрутизация → Анализ знаний → Контекст-пак → LLM → ответ
```

### Пайплайн скиллов

```
Сообщение → Skill Router
├── Активная сессия? (RPG) → Продолжить скилл → handler.process_message()
├── Триггер-слово? → Активировать скилл → handler.process_message()
├── LLM классификация? → Если совпало → handler.process_message()
└── Нет → Обычный пайплайн Псины
```

### Пайплайн памяти

```
Лог (БД) → Extraction (батчи, confidence ≥ 0.6) → Structured Memory (факты + embeddings)
→ Retrieval (vector + keyword) → Context Pack (лимитированный) → LLM
```

Псина **читает все сообщения**, но **отвечает только** когда:
- Позвали по имени (Псина, пёс, пёсик)
- Ответили на её сообщение
- Находятся в активной сессии (RPG)
- Активирован скилл (заметки, напоминания, поиск)

## Гибкость и расширяемость

### Как добавить новый скилл

Скиллы следуют стандарту **SKILL.md** — markdown-файл с YAML frontmatter. **Никакой регистрации не нужно** — бот обнаруживает скиллы автоматически при старте.

Система поддерживает **два режима исполнения**:

#### PromptOnlySkill — только инструкции

```
src/skills/my_skill/
├── __init__.py       # пустой
└── SKILL.md          # имя, описание, инструкции
# handler.py НЕ НУЖЕН — используется дефолтный
```

#### ExecutableSkill — инструкции + кастомная логика

```
src/skills/my_skill/
├── __init__.py       # пустой
├── SKILL.md          # имя, описание, инструкции
└── handler.py        # async def process_message(msg, chat_id, user_id) -> str | None
```

**SKILL.md формат:**
```yaml
---
name: My Skill
description: >
  Краткое описание (до 1024 символов) — по нему LLM-роутер решает
  когда использовать этот скилл. Пиши конкретно: фразы-триггеры,
  примеры запросов, что скилл умеет.
license: MIT
compatibility: "1.0"
---

# Instructions

Подробные правила поведения скилла для LLM...
```

**Как это работает (Progressive Disclosure):**
1. **Discovery** — при старте читается только `name` + `description` (~50-100 токенов каждый)
2. **Activation** — когда задача совпадает с `description`, загружается полный SKILL.md
3. **Execution** — если есть `handler.py` — вызывается он, иначе — дефолтный handler

**Типы скиллов:**

| Скилл | Режим | Зачем свой handler |
|---|---|---|
| **RPG Game Master** | ExecutableSkill | Броски, Session Zero, мульти-сессии, состояние |
| **Заметки** | ExecutableSkill | CRUD в SkillState, LLM-классификация операций |
| **Напоминания** | ExecutableSkill | LLM-парсинг времени, запись в таблицу `reminders` |
| **Переводчик** | PromptOnlySkill | Достаточно инструкций «ты переводчик EN→RU» |
| **Погода** | ExecutableSkill | Вызов внешнего API |

## Быстрый старт

### Docker Compose (рекомендуется)

```bash
# 1. Клон
git clone <repo>
cd Zalutka

# 2. Конфиг
cp .env.example .env
# Заполни TELEGRAM_BOT_TOKEN и LLM_API_KEY

# 3. Запуск
docker compose up -d --build

# 4. Миграции
docker compose exec bot alembic upgrade head
```

### Локально

```bash
# 1. Запуск зависимостей
cp .env.example .env
docker compose up -d db redis

# 2. Установка зависимостей
pip install -e .

# 3. Миграции
DATABASE_URL_SYNC=postgresql+psycopg2://postgres:postgres@localhost:5432/zalutka \
  alembic upgrade head

# 4. Запуск бота
python -m src.main
```

### Без LLM (mock режим)

```env
LLM_PROVIDER=mock
```

Бот работает, но отвечает заглушками. Для полноценной работы нужен OpenRouter, Qwen или другой OpenAI-совместимый LLM.

FastAPI на `127.0.0.1:8000`. Все эндпоинты требуют Bearer-токен (`ADMIN_API_SECRET`).

| Эндпоинт | Метод | Описание |

| `/api/overview` | GET | Обзор системы (юзеры, память, скиллы, статус БД/Redis) |
| `/api/skills/` | GET | Список скиллов |
| `/api/skills/agent_rpg/sessions?chat_id=X` | GET | RPG-сессии чата |
| `/api/skills/agent_rpg/sessions/{id}?chat_id=X` | DELETE | Удалить RPG-сессию |
| `/api/skills/agent_notes/notes?chat_id=X` | GET | Заметки чата |
| `/api/skills/agent_notes/notes/{id}?chat_id=X` | DELETE | Удалить заметку |
| `/api/skills/agent_reminders/active?chat_id=X` | GET | Активные напоминания |
